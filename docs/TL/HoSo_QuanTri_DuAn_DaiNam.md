## 🎬 NARRATIVE — "CODE XONG RỒI VIẾT TÀI LIỆU NGƯỢC"

> **Bối cảnh thực tế:** Team đã code xong toàn bộ hệ thống, bây giờ cần tạo tài liệu Jira/Confluence
> mô phỏng quá trình phát triển Agile/Scrum cho môn Triển Khai Phần Mềm.

### Câu chuyện kịch bản:

1. **Sprint 1** — Team **tự thiết kế** DB v1 (`dbCu.sql` — 22 bảng) theo phân tích yêu cầu ban đầu. Code 7 form CRUD cơ bản. Mọi thứ chạy ổn.

2. **Sprint 2** — Mở rộng 14 form phức tạp (KS, NH, Kho, Ví...) → **Phát hiện DB v1 sai hướng**: không scale cho tài chính phức tạp, không có bảng đoàn khách, BUS spaghetti. 3 Bug CRITICAL blocked.

3. **Sprint 3 (TÂM BÃO)** — PO cũng feedback UI không phù hợp POS. Team **cắn răng đập đi xây lại** 100%:
   - DB: 22 → 60+ bảng (3 CR được duyệt)
   - Architecture: DAL.Instance → Gateway + DI + MVP
   - UI: Traditional → POS cảm ứng
   - Bổ sung Loyalty (scope creep chấp nhận)

4. **Sprint 4** — Backend vững → Mở Web B2C (Blazor + Kestrel). Chặn hacker HTTP inject.

5. **Sprint 5** — DevOps (`run.bat` + `deploy.bat`). Seed data. Tài liệu. Submit.

### Điểm hay để trình bày trong môn học:
- ✅ Có **Change Request** thực tế (3 CR với Impact Analysis)
- ✅ Có **Technical Debt** phát sinh → giải quyết bằng Refactoring Sprint
- ✅ Có **Bug lifecycle** (Found → Blocked → Fixed)
- ✅ Có **Velocity fluctuation** (25 → 30 → 61 → 23 → 18) — giải thích được spike Sprint 3
- ✅ Có **ADR** (Architecture Decision Records) — trình bày trade-offs
- ✅ Có **DB Migration Analysis** — so sánh 3 version schema
- ✅ Có **Deployment Manual** — O2O kiến trúc thực tế
